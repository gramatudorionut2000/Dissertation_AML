About

We provide machine learning models to be used by (simulated) auditors in detecting money laundering activities, as well as functionalities for model explainability and transaction visualization

Core Features

Network Visualization

Visualize transactions as networks. Analyze how accounts are connected through transaction flows.

Machine learning AML

Machine learning algorithms for detecting laundering patterns in inter-institutional financial data.

Data Importing

Import transaction data formatted according to IBM financial data structure.

Filtering

Apply filters to the data based on the features to better focus the analysis.

Transaction Visualization

As the current implementation is meant to simulate the activity of a financial auditor, the users are able to visualize the transactions a node (an account) is implicated in, up to a depth of 6 nodes, which are represented as subgraphs.

Data Loading Page

By either uploading a file or providing a local path for a data file, the user is to load a CSV file with the data (it has to respect the structure of the IBM data) into the database. After the database is populated, the user will be able to use pages where it can visualize the data, as well as conduct model training and inference.

Money Laundering Prediction

After loading in the data, the user will be able to use a training page, where they can train various models on the preprocessed data, or use checkpoints of pretrained models in order to conduct inference and predict whether a transaction is fraudulent or not

Step 1: Data Preprocessing

Preprocess tabular transaction data into features that can be used by machine learning models

Step 2: Model Training

Train machine learning models on preprocessed data or load pretrained model checkpoints

Step 3: Fraud Prediction

Use models on transactions in order to identify potentially fraudulent transactions

Software Application Specifications

  • Backend built using Djnago
  • Machine learning models implemented using Pytorch and Pytorch-Geometric
